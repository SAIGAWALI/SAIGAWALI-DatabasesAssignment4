[30 seconds]
"Let's talk about the trade-offs. Sharding gives us three major wins:

First, THROUGHPUT triples. A single server handles 1,000 requests per 
second. Three shards handle 1,000 each = 3,000 total.

Second, LATENCY drops 2 to 3 times. Before sharding: 50 to 100 milliseconds 
because all queries compete for locks. After sharding: 15 to 30 milliseconds 
because each shard has less contention and better cache hits.

Third, STORAGE triples. One terabyte becomes three terabytes."


[45 seconds]
"But we made trade-offs. The CAP theorem says you can pick 2 out of 3: 
Consistency, Availability, or Partition tolerance. We picked Consistency 
and Partition tolerance.

Why? Because this is healthcare. Medical data must be ACCURATE. If there's 
a network split between shards, we guarantee correct data even if some 
shard goes temporarily offline. We don't want a situation where Shard 1 
thinks a patient has medication A but Shard 2 thinks they have medication B.

That's unacceptable for healthcare. Accuracy > uptime."


[30 seconds]
"What did we sacrifice? Operational complexity increased. We now manage 
3 databases instead of 1. Queries became more complex—cross-shard joins 
require application-layer coordination. Infrastructure cost tripled.

But that's the correct trade-off for a healthcare system at scale. When 
you have 1,000 patients, 10,000 appointments, and growing, you NEED 
sharding or the single database collapses.

That's why sharding is the right choice for this assignment."

==========================================================================================
CAP THEOREM: Pick 2 out of 3
==========================================================================================

  Consistency (C)    : All clients see the same data
  Availability (A)   : System is always available
  Partition (P)      : System tolerates network failures

Three possible choices:
  1. CA  - Consistency + Availability  (No partition tolerance)
  2. CP  - Consistency + Partition     (May be unavailable)
  3. AP  - Availability + Partition    (May have stale data)

OUR CHOICE: CP (Consistency + Partition Tolerance)

🎯 WHY WE CHOSE CP:

  1. Healthcare Data
     → Medical records MUST be accurate. Stale data = patient harm.

  2. Compliance
     → HIPAA/regulations require data consistency, not 100% uptime.

  3. Trade-off
     → Prefer correct data with occasional downtime vs wrong data always available.

  4. Acceptable Risk
     → Network partitions rare in datacenter. Single shard outage = acceptable.
